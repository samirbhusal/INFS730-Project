Technical Notes:

1. Add a UserStatus column to the users table with values such as Pending and Active.

2. When Admin creates a new employee account:
   - insert the employee as Pending
   - generate a secure one-time cryptographic token
   - store only the hashed token in the database
   - send the plain activation link to the user by email
   - set the link to expire after 24 hours

3. Activation logic:
   - validate that the token exists
   - validate that it is unused
   - validate that it is not expired
   - only after successful password creation should the system:
     - hash the password securely
     - save the password hash
     - change UserStatus from Pending to Active
     - mark the token as used

4. Security:
   - passwords must never be stored in plain text
   - use password_hash() / password_verify()
   - token must be one-time-use
   - expired or used links must show a Link Invalid page
